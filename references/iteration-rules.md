# Iteration Rules

Use this when the user provides Qianchuan campaign or creative performance data.

## Required Data

Useful fields:

- Creative name
- Spend
- Impressions
- CTR
- 3s/5s retention or average watch time
- Product click rate or live-room entry rate
- CVR
- CPA
- ROI
- Comments or negative feedback

If exact metrics are unavailable, work with relative descriptions but mark conclusions as weak.

## Diagnosis

| Symptom | Likely issue | Next creative action |
|---|---|---|
| Low CTR | Hook/cover/first frame weak | Test sharper hook, clearer product, stronger pain scene |
| Good CTR, low retention | Opening overpromises or pacing weak | Align first 3s with actual content, shorten setup |
| Good retention, low click | Selling point not tied to purchase reason | Add offer, proof, comparison, or clearer CTA |
| Good click, low CVR | Landing/live-room承接 or price trust issue | Align script with live-room/product page, improve proof |
| High CPA, decent CTR | Broad curiosity but weak buyer intent | Shift from curiosity hook to qualified pain/persona |
| High negative comments | Claim mismatch or trust issue | Reduce absolute claims, add proof, change tone |
| One angle wins repeatedly | Winning audience-pain fit | Generate variants on hook/speaker/material while preserving core angle |

## Iteration Output

When analyzing data, output:

1. Winning patterns
2. Losing patterns
3. Claim or compliance risks observed
4. Next 5 creative hypotheses
5. Scripts or hooks for the next batch
6. What data to collect in the next test

## Guardrails

Do not recommend scaling solely from CTR. Treat ROI/CPA and conversion path quality as decisive for budget decisions.
